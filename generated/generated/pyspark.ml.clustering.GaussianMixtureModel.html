pyspark.ml.clustering.GaussianMixtureModel¶

class pyspark.ml.clustering.GaussianMixtureModel(java_model=None)[source]¶

Note

Experimental

Model fitted by GaussianMixture.

New in version 2.0.0.

__init__(java_model=None)¶

Initialize this instance with a Java model object. Subclasses should call this constructor, initialize params, and then call _transfer_params_from_java.

This instance can be instantiated without specifying java_model, it will be assigned after that, but this scenario only used by JavaMLReader to load models. This is a bit of a hack, but it is easiest since a proper fix would require MLReader (in pyspark.ml.util) to depend on these wrappers, but these wrappers depend on pyspark.ml.util (both directly and via other ML classes).
